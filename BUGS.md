# Bug H.1

| Параметр | Значение |
|----------|----------|
| **Описание** | API принимает отрицательные значения цены (price < 0) и успешно создает объявление вместо возврата ошибки валидации |
| **Проект** | QA-trainee-assignment-autumn-2025 (Задание 2.1) |
| **Приоритет (Priority)** | High |
| **Эндпоинт** | POST /api/1/item |
| **Ожидаемый результат** | Статус 400 с сообщением об ошибке валидации отрицательного значения цены |
| **Полученный результат** | Статус 200, объявление успешно создается с отрицательной ценой |
| **Шаги воспроизведения** | Выполнить POST запрос на /api/1/item с телом:<br>```json<br>{<br>  "sellerId": 123456,<br>  "name": "Product with negative price",<br>  "price": -100<br>}<br>``` |
| **Тесты связанные с багом** | test_create_item_negative_price |
